# 資産インフレ対策案

現在、プレイヤー資産が何十億円、何百億円まで膨らんでいます。これは単に「税金を上げる」だけでは解決しにくいです。強いプレイヤーほど資産が増え続け、弱いプレイヤーだけが苦しくなる可能性があるからです。

解決の方向性は、次の3つです。

1. 増えすぎたお金を回収する
2. お金を使いたくなる大型コンテンツを増やす
3. 株価・資産の伸び方をなだらかにする

## 現状の問題

| 問題 | 起きていること |
|---|---|
| 株価上昇の影響が大きい | 保有株が跳ねると資産が一気に増える |
| お金の使い道がまだ少ない | 億単位の資産に対して費用が軽い |
| 税金が現金からしか引かれない | 株で資産を持つプレイヤーは逃げやすい |
| ログインボーナスが固定 | 初心者には有効だが、富豪には意味が薄い |
| 大富豪向けのリスクが少ない | 一度勝つと逆転されにくい |

## すぐ入れたい対策

まずは、今のゲームを壊しにくい対策から入れるのが安全です。

| 対策 | 内容 | 効果 |
|---|---|---|
| 累進資産税 | 総資産が大きいほど税率が上がる | 富豪だけを狙って回収できる |
| 維持費 | 自社ステータスが高いほど定期費用がかかる | 会社強化の放置勝ちを防ぐ |
| 富豪向け高額アクション | 数億円単位の使い道を作る | お金を燃やす楽しい出口になる |
| 株価上限の圧縮 | 高すぎる株価ほど伸びにくくする | 億万長者の増殖速度を抑える |
| ランキング報酬を称号中心に | 現金報酬を控える | インフレを増やさず競争できる |

## 累進資産税

今の税金は総資産10%ですが、全員一律だと初心者も苦しくなります。資産帯ごとに税率を変えた方がよいです。

| 総資産 | 税率 |
|---:|---:|
| 0円から1,000万円 | 0% |
| 1,000万円から1億円 | 2% |
| 1億円から10億円 | 5% |
| 10億円から100億円 | 10% |
| 100億円以上 | 18% |

ポイントは、初心者や普通のプレイヤーは軽く、大富豪はしっかり取ることです。

現金が足りない場合は、いきなり株を没収するとつらいので、まずは「未払い税金」として記録するのがよいです。

```json
{
  "unpaidTax": 125000000
}
```

未払い税金がある間は、以下を制限できます。

| 制限 | 内容 |
|---|---|
| 新規会社設立不可 | まず納税が必要 |
| M&A不可 | 富豪の暴走を止める |
| 提携不可 | 経済活動には納税が必要 |
| ランキングに滞納表示 | ちょっとした圧になる |

## 富豪向け高額アクション

お金が余っているなら、税金で取り上げるだけでなく「使いたくなる浪費」を作る方が楽しいです。

| アクション | 費用 | 効果 |
|---|---:|---|
| 世界広告キャンペーン | 5億円 | ブランドと注目度を大きく上げる |
| 研究所建設 | 10億円 | 技術力の上限を上げる |
| 本社ビル建設 | 20億円 | 称号と信頼を得る |
| 宇宙事業参入 | 50億円 | 成功すれば大ニュース、失敗で大損 |
| 巨額配当 | 10億円 | 株主人気を大きく上げる |
| 国家プロジェクト入札 | 30億円 | 成功で財務・信頼・技術力アップ |
| スーパー買収防衛 | 15億円 | 24時間買収されにくくなる |

こういう「富豪専用コンテンツ」があると、資産が増えすぎてもゲーム内で燃やせます。

## 自社維持費

会社ステータスが高いほど維持費を取ると、強い会社を維持する意味が出ます。

計算例:

```js
maintenanceCost =
  (technology + brand + trust + finance + morale + attention) * 50000;
```

ただし、資産が少ないプレイヤーには厳しいので、維持費は会社を持っているプレイヤーだけ、かつ最低資産以上にします。

| 条件 | 維持費 |
|---|---|
| 総資産1,000万円未満 | なし |
| 総資産1,000万円以上 | 自社ステータス合計に応じて徴収 |
| 総資産10億円以上 | 維持費倍率を上げる |

## 株価伸びすぎ対策

株価が高くなるほど上がりにくくします。

例:

```js
const pricePressure = Math.min(0.8, currentPrice / 100000000);
newPrice = prev * (1 + drift * (1 - pricePressure));
```

これで、低価格株は動きやすく、高価格株は伸びにくくなります。

また、暴騰しすぎた銘柄には自動でイベントを入れるのもよいです。

| 条件 | イベント |
|---|---|
| 1時間で+100%以上 | 過熱感警戒 |
| 1日で+500%以上 | 取引規制 |
| 株価100万円超え | 分割検討 |

## 株式分割

株価が高すぎる銘柄は、自動で株式分割できます。

例:

| 条件 | 処理 |
|---|---|
| 株価100万円超え | 1株を10株に分割 |
| 保有株数 | 10倍 |
| 株価 | 10分の1 |
| 総評価額 | 変わらない |

これはインフレ対策というより、表示と売買のしやすさ改善です。

## ログインボーナスの見直し

固定1万円は初心者には助かりますが、富豪には意味がありません。資産帯で変えるとよいです。

| 総資産 | ボーナス |
|---:|---:|
| 100万円未満 | 5万円 |
| 1,000万円未満 | 2万円 |
| 1億円未満 | 1万円 |
| 1億円以上 | なし |

これで初心者救済として機能し、大富豪への追加注入を止められます。

## M&A費用の見直し

買収費用は、相手株価だけでなく相手の総評価や防衛力に応じてもっと重くします。

おすすめ計算:

```js
takeoverCost =
  targetPrice * 3000
  + targetDefenseLevel * 50000000
  + targetTrust * 10000000;
```

今よりさらに高くして、M&Aは「富豪の大勝負」にした方がよいです。

## 支援・提携の悪用対策

資金支援は良い機能ですが、複数アカウントで資金移動されるとインフレを助長します。

対策:

| 対策 | 内容 |
|---|---|
| 支援手数料 | 送金額の20%を市場に消す |
| 1日上限 | 1人あたり1日1,000万円まで |
| 同一相手制限 | 同じ相手へ1日1回まで |
| 支援履歴表示 | 透明性を上げる |
| 新規アカウント制限 | 登録直後は支援不可 |

資金支援は「仲良くする機能」ですが、経済的にはお金を増やさず、むしろ少し減るようにした方が安全です。

## 借金・破産システム

大富豪対策とは逆に、失敗したプレイヤーの復帰策も必要です。

| 機能 | 内容 |
|---|---|
| 破産申請 | 資産が一定以下なら所持株を清算して再スタート |
| 再建ボーナス | 破産後に少額資金を付与 |
| 破産称号 | ペナルティではなくネタとして表示 |
| 債務整理 | 未払い税金を一部免除 |

破産があると、失敗を恐れずに大勝負しやすくなります。

## おすすめ実装順

一番効きそうな順番です。

1. 累進資産税
2. ログインボーナスの資産連動化
3. 支援手数料と支援上限
4. 富豪向け高額アクション
5. 自社維持費
6. M&A費用の再調整
7. 株価伸びすぎ補正
8. 株式分割
9. 未払い税金と制限
10. 破産・再建システム

## 最初に実装するなら

最初の小さなアップデートとしては、これがよいです。

| 項目 | 仕様 |
|---|---|
| 累進資産税 | 1,000万円未満0%、1億未満2%、10億未満5%、100億未満10%、100億以上18% |
| ログインボーナス | 1億円以上はなし |
| 資金支援手数料 | 20%を消滅、80%だけ相手に届く |
| 支援上限 | 1日1,000万円 |
| 富豪アクション | 5億円以上の高額アクションを3つ追加 |

これだけで、資産インフレはかなり落ち着くはずです。

## 方針

資産インフレの対策は、罰だけにしない方がいいです。

強いプレイヤーからお金を回収しつつ、そのお金を使って「かっこいいこと」「目立つこと」「危険な大勝負」ができるようにすると、ゲームとして気持ちよく調整できます。
